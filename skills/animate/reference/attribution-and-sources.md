# Attribution and sources

The motion doctrine in `skills/animate/` is informed in part by [Emil Kowalski's animations.dev course](https://animations.dev/).

The course teaches both the theory and practice behind great web animation, with a strong emphasis on:

- choosing easing and duration intentionally
- understanding why a motion decision feels off
- knowing when motion helps and when it should stay out of the way
- building tasteful, performant animation with CSS and Motion / Framer Motion

The course site identifies Emil as the teacher and also highlights his work on [Sonner](https://github.com/emilkowalski/sonner) and [Vaul](https://github.com/emilkowalski/vaul), two React UI libraries where motion quality is part of the product feel.

This repository uses that motion lineage as inspiration, but rewrites it into original guidance tailored to `better-web-ui`.

Use this attribution reference when:

- documenting where the `animate` skill's motion taste comes from
- explaining why certain drawer, toast, easing, or spring defaults appear in the library
- keeping attribution explicit without duplicating course material
