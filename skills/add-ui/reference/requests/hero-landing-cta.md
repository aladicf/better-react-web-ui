### Hero / landing / CTA

Prioritize:

- hierarchy first
- strong typography
- believable proof
- one obvious primary action

Also:

- define one primary audience or buying situation; trying to speak to everyone usually weakens the page for everyone
- match the page promise to the user's likely traffic source and awareness level
- define the value proposition clearly as both the benefit and how the product delivers it
- sharpen the USP from the customer's point of view; translate features into benefits and keep only the differentiators that matter for this audience
- make the headline outcome-specific, not category-foggy
- make the heading, subheading, and CTA work as one unit: the heading hooks, the subheading adds specifics, and the CTA states the next step plainly
- if a hero headline or subhead wraps, reduce the type size before crushing the leading; multi-line hero text needs enough line-height and top/bottom padding that glyphs never clip at any breakpoint
- do not default to an abstract full-bleed hero image when product clarity, proof, or a tighter product visual would answer the page's first questions better
- keep hero copy in a quiet part of the image and off faces, product details, or busy focal points; if no safe text zone exists, change the crop, the asset, or the hero strategy
- break long landing pages into clearly separated logical blocks instead of letting headings, proof, and CTAs melt into one continuous slab
- distinguish credibility, proof, and testimonial jobs instead of collapsing them into one generic trust wall
- keep proof close to the strongest claim
- use a short 3–4 step explanation when the service or workflow needs demystifying
- use CTA text that says what the user gets next, not generic button filler
- handle predictable objections near the relevant CTA, form, or proof point instead of hiding them elsewhere
- keep a secondary CTA quieter and lower-commitment when one is needed
- treat footer/contact content as part of the conversion surface: it should make support, legitimacy, and next steps easy to verify without becoming clutter
- add blogs, resources, downloads, or FAQs only when they answer real user questions or strengthen the offer; do not keep them as decorative filler
- if a hero uses a carousel, make sure the first panel carries the strongest message and that the content remains reachable elsewhere instead of relying on later slides for discovery

For redesigns of an existing hero, landing page, blog page, login/signup page, or CTA surface:

- preserve the current narrative skeleton when the user asks to keep the essence: section order, proof blocks, CTA placement, and major content zones should usually stay recognizable
- push distinction through typography, palette, copywriting tone, proof styling, imagery direction, spacing rhythm, and detail language before changing the information architecture
- if you offer a stretch direction, label it clearly as the bolder structural departure instead of treating it as the default
